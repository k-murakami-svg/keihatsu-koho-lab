# 10. Claude Code 実装依頼プロンプト

このファイルを最初にClaude Codeに読ませることで、プロジェクト全体を把握させます。

---

## Claude Code への依頼文（コピペ用）

以下の文章をそのままClaude Codeに渡してください：

---

```
啓発広報ラボのWebサイトをNext.js + microCMSで実装してください。

【最初に必ず読むべきドキュメント】
docs/ フォルダにある以下のMDファイルを、順番に読んでから実装を開始してください：

1. 01-project-overview.md（プロジェクト概要・技術スタック）
2. 02-design-system.md（デザインシステム詳細）
3. 03-page-copy-top.md（トップページ文言）
4. 04-page-copy-other.md（他ページ文言）
5. 05-seo-geo-config.md（SEO/GEO設計）
6. 06-structured-data.md（構造化データJSON-LD）
7. 07-microcms-schema.md（microCMSスキーマ）
8. 08-faq-content.md（FAQ全文）
9. 09-implementation-steps.md（実装手順）

【プロジェクト概要】
- サイト名：啓発広報ラボ
- 運営：株式会社アトムストーリー
- 目的：業界団体・公益法人の啓発広報を支援する専門メディア兼サービスサイト
- 技術：Next.js 14（App Router）+ Tailwind CSS + microCMS + Vercel
- ドメイン：keihatsu-koho-lab.jp

【実装の進め方】
Phase 1：基本ページ（トップ・サービス・事例詳細・お問い合わせ）
Phase 2：追加ページ（会社概要・代表メッセージ・ナレッジ・インタビュー）
Phase 3：SEO/GEO対策（構造化データ・サイトマップ・GA4・Search Console）

【最重要の指示】
1. ドキュメントに記載された配色（#1E3A5F、#D4A437、#FFFFFF、#2B2B2B）を厳密に使うこと
2. フォントは Noto Serif JP / Noto Sans JP / Inter のみ使うこと
3. 各ページのtitleとdescriptionは 05-seo-geo-config.md の通りに設定すること
4. 構造化データ7種類は 06-structured-data.md のコードを正確に実装すること
5. レスポンシブ対応必須（ブレークポイント768px）
6. アクセシビリティに配慮（セマンティックHTML、alt属性、ARIA属性）

【まず最初に質問してほしいこと】
1. microCMSのアカウントは作成済みか？スキーマは設定済みか？
2. Vercelアカウントは作成済みか？
3. ドメインは取得済みか？
4. Resend（フォーム送信）のAPIキーは取得済みか？
5. デザインモックアップ画像はあるか？

これらが揃っていない場合は、揃えるための手順を案内してください。
揃っている場合は、Phase 1から実装を開始してください。

不明点があれば、実装開始前に質問してください。
段階的に進め、各Phaseの完了時に状態を報告してください。
```

## このプロンプトの使い方

### Step 1：プロジェクトディレクトリの準備

```bash
# 任意の場所にプロジェクトディレクトリを作成
mkdir keihatsu-koho-lab
cd keihatsu-koho-lab

# docsフォルダを作成し、MDファイル群を配置
mkdir docs
# MDパッケージの全ファイル（01〜10）をdocs/にコピー
```

### Step 2：Claude Codeを起動

```bash
# Claude Codeをインストール（初回のみ）
# https://docs.claude.com/claude-code/

# Claude Codeを起動
claude
```

### Step 3：上記のプロンプトを貼り付け

このファイルの「Claude Code への依頼文」セクションをコピーして、Claude Code に貼り付けます。

### Step 4：質問に回答しながら進める

Claude Codeが質問してくるので、答えながら進めます：
- 「microCMSのアカウントは作成済みか？」→ 作成済み/未作成で回答
- 「Vercelアカウントは作成済みか？」→ 同様
- など

### Step 5：実装を確認しながら進める

Claude Codeが段階的に実装します。各フェーズで動作確認しながら進めます：
- Phase 1完了 → ローカルで動作確認 → Phase 2へ
- Phase 2完了 → ローカルで動作確認 → Phase 3へ
- Phase 3完了 → Vercelデプロイ

## 追加で必要な情報

実装中に、Claude Codeが以下を質問してくる可能性があります。事前に準備しておくとスムーズです：

### 1. お問い合わせフォームの送信先メールアドレス
```
例：kenta@atom-story.com
※ Resendの設定にも必要
```

### 2. Googleアナリティクスの測定ID
```
例：G-XXXXXXXXXX
※ Google Analytics の管理画面で確認
```

### 3. デザインモックアップ画像
```
ChatGPTで作成したモックアップ画像（複数枚）
※ Claude Codeに参考として渡す
```

### 4. 各種ロゴ・画像素材
```
■ ロゴ画像（logo.png / SVG）
■ OGP画像（1200×630px）
■ 家電製品協会様の動画埋め込みURL
■ その他必要な画像
```

これらが揃っていない場合、Claude Codeはプレースホルダー画像で実装することも可能です。後から差し替えれば問題ありません。

## デザインモックアップ画像を渡す場合

Claude Codeに以下を伝えてください：

```
ChatGPTで作成したデザインモックアップ画像を添付します。

【重要な注意】
・これらの画像は「レイアウトとビジュアルの参考」として使用してください
・配色・余白・フォントサイズは、必ず docs/02-design-system.md の数値を厳守してください
・画像内のテキストではなく、docs/03-page-copy-top.md および docs/04-page-copy-other.md の文言を使用してください

画像が参考にすべき要素：
・全体のレイアウト構造
・要素の配置
・ビジュアルの雰囲気

画像から無視すべき要素：
・正確な配色（仕様書を優先）
・正確な余白（仕様書を優先）
・テキストの内容（文言ファイルを優先）
```

## トラブル時の対応

### Claude Codeが質問してきた場合
→ docs/フォルダの該当ファイルを参照しつつ、回答してください。

### Claude Codeが実装を間違えた場合
→ 該当する仕様書（01〜10のMDファイル）を再度参照させ、修正を依頼してください。
例：「02-design-system.mdの配色仕様を確認して、CSSの色を修正してください」

### 仕様の追加・変更が必要な場合
→ 該当MDファイルを直接編集してから、Claude Codeに「仕様書を更新したので確認してください」と伝えてください。

## このパッケージの強み

```
■ 設計と実装の完全分離
　・設計：MDファイル群（人間が編集可能）
　・実装：Claude Codeが生成

■ 修正が容易
　・仕様変更があってもMDファイルを編集するだけ
　・Claude Codeに再依頼で対応可能

■ Git管理可能
　・docs/フォルダごとリポジトリ管理
　・変更履歴が追跡可能

■ 他の人にも引き継ぎ可能
　・将来別のエンジニアに依頼する場合も、このパッケージで完結
```

## 最後に

啓発広報ラボWebサイトの実装は、このMDパッケージで必要な情報がほぼすべて揃っています。

Claude Codeに依頼する際は、**MDファイル群を必ず読ませる**ことを最初に指示してください。
