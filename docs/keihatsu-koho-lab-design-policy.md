# 啓発広報ラボ デザイン方針指示書

**最終確定日**：2026年6月
**対象**：Claude Code での実装、外注先への共有、社内共有

このドキュメントは、啓発広報ラボWebサイトのデザイン方針を1枚に集約した最終版です。

---

## 1. 全体トーン

**「温かみのあるプロフェッショナル」**

業界団体・公益法人の意思決定者が「ちゃんとした専門会社」と感じ、かつ「相談していい」と思える、両立したトーン。

### 軸

```
■ プロフェッショナル軸
　・清潔感
　・誠実
　・洗練
　・知的
　・信頼感

■ 温かみ軸
　・寄り添う
　・温かい
　・人の手を感じる
　・親しみ
```

### 避けるべき印象

```
✗ 冷たい・無機質（プロに振りすぎ）
✗ カジュアル・軽い（温かみに振りすぎ）
✗ 派手・カラフル
✗ 堅苦しい・古い
✗ スタートアップ・SaaS的なカラフルさ・グラデーション
```

## 2. 配色（確定版）

### カラーパレット

| 役割 | 色名 | HEX | RGB | 使用比率 |
|---|---|---|---|---|
| ベース | ピュアホワイト | `#FFFFFF` | 255, 255, 255 | 70% |
| メイン | ディープネイビー | `#1E3A5F` | 30, 58, 95 | 25% |
| **アクセント** | **マスタード** | **`#D4A437`** | **212, 164, 55** | **5%** |
| テキスト | 濃いグレー | `#2B2B2B` | 43, 43, 43 | 本文 |
| 罫線 | ライトグレー | `#E0E0E0` | 224, 224, 224 | UIパーツ |
| 強調背景 | 薄いベージュ | `#FAF8F2` | 250, 248, 242 | 必要に応じて |

### アクセントカラー（マスタード #D4A437）の役割

```
■ 使用箇所
　・メインCTAボタン（塗り）
　・セクション区切り線（短い装飾線）
　・強調マーカー
　・リンクのホバー時の色
　・FAQの「Q.」「A.」の接頭辞
　・丸数字バッジ

■ 使用比率
　・全体の5%程度に抑える
　・「ここぞ」という箇所で使う
　・多用するとブランド印象が弱まる

■ 選定理由
　・温かみと知的さの両立
　・業界団体・公益法人の意思決定者層に違和感がない色
　・手描き風イラストとの相性が良い
　・ネイビーとのコントラストが綺麗
```

### 各色の使い分けルール

```
■ ネイビー（#1E3A5F）
　・見出し（H1〜H3）
　・ヘッダー帯
　・フッター
　・メインキャッチの文字色
　・サブCTAボタンの枠線・文字
　・本文中の重要文字

■ マスタード（#D4A437）
　・メインCTAボタン（塗り）
　・セクション区切り線
　・強調マーカー
　・リンクのホバー時の色

■ 白（#FFFFFF）
　・ページ背景
　・カード背景
　・ネイビー帯の上の文字

■ 濃いグレー（#2B2B2B）
　・本文テキスト全般
　・小見出し（H4以下）

■ ライトグレー（#E0E0E0）
　・カード罫線
　・区切り線

■ 薄いベージュ（#FAF8F2）
　・強調エリアの背景
　・クライアントコメント枠
　・FAQセクションの背景など
```

## 3. タイポグラフィ

### フォント構成

| 用途 | フォント | 種類 |
|---|---|---|
| 見出し（大） | Noto Serif JP Bold | 明朝 |
| 小見出し・本文 | Noto Sans JP | ゴシック |
| 英数字 | Inter | サンセリフ |

### 選定理由

```
■ Noto Serif JP（見出し）
　・明朝の知的さで「ちゃんとした感」を出す
　・業界団体・公益法人向けに馴染む
　・無料、Google Fonts で読み込み可能

■ Noto Sans JP（本文）
　・読みやすい
　・どの環境でも安定表示
　・無料、Google Fonts

■ Inter（英数字）
　・モダンで洗練された印象
　・数字が読みやすい
　・無料、Google Fonts
```

### Google Fonts 読み込み

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;500;700&family=Noto+Serif+JP:wght@700&family=Inter:wght@400;500;600;700&display=swap" rel="stylesheet">
```

### 用途別のフォント・サイズ指定

```
■ ヒーローのメインキャッチ
　・フォント：Noto Serif JP Bold（明朝）
　・色：ネイビー（#1E3A5F）
　・サイズ：PC 44px〜52px / スマホ 28px〜32px
　・例：「啓発に、ちゃんと手応えを。」

■ セクションのメインキャッチ
　・フォント：Noto Serif JP Bold（明朝）
　・色：ネイビー（#1E3A5F）
　・サイズ：PC 32px〜36px / スマホ 22px〜24px
　・例：「私たちが提供する3つの支援」

■ 小見出し
　・フォント：Noto Sans JP Bold
　・色：ネイビー（#1E3A5F）または マスタード（#D4A437）
　・サイズ：PC 20px〜24px / スマホ 18px〜20px

■ 本文
　・フォント：Noto Sans JP Regular
　・色：濃いグレー（#2B2B2B）
　・サイズ：PC 16px / スマホ 15px
　・行間：1.7

■ キャプション・補足
　・フォント：Noto Sans JP Regular
　・色：グレー（#666666）
　・サイズ：PC 13px / スマホ 12px

■ 英数字（数字・年号など）
　・フォント：Inter
　・色：状況に応じて
　・サイズ：本文と同じ
```

## 4. レイアウト

### グリッドシステム

```
■ PC最大幅：1200px
　・コンテナ：max-w-[1200px] mx-auto px-6
　・左右の余白：自動センタリング

■ ブレークポイント
　・md: 768px
　・768px以下：スマホレイアウト
　・768px以上：PCレイアウト
```

### 間隔（スペーシング）

```
■ セクション間隔
　・PC：80px（py-20）
　・スマホ：60px

■ セクション内の要素間隔
　・PC：32px〜48px
　・スマホ：24px〜32px

■ 行間（line-height）
　・本文：1.7
　・見出し：1.4
```

### カラム構造

```
■ 基本：1カラム（読み物として上から下へ）

■ 例外
　・3つの支援セクション：3カラム
　・事例セクション：2カラム可
　・ナレッジ記事一覧：3カラム or 2カラム

■ スマホ：すべて縦積み（1カラムに変換）
```

## 5. 素材方針

### 写真

```
■ 使用箇所（限定的）
　・事例ページ
　・代表メッセージページ
　・インタビュー記事

■ 使わないもの
　・フリー素材写真（信頼性を損なう）
　・トップページのヒーロー写真
```

### イラスト

```
■ 方針
　・パラパラ漫画と同じテイストの手描き風イラスト
　・サイト全体に控えめに散らす（あしらいレベル）

■ 色
　・モノクロまたはマスタード単色
　・カラフルにしすぎない

■ 使用箇所例
　・トップページのヒーロー横
　・3つの支援セクションのアイコン代わり
　・「パラパラ漫画が効く4つの理由」の各カード
　・セクション区切りのアクセント
```

### あしらい・装飾

```
■ 使うもの
　・細い罫線（マスタード または ライトグレー）でセクション区切り
　・余白で構造を作る
　・薄いベージュ背景（強調エリア）

■ 使わないもの
　・背景パターン
　・背景グラデーション
　・派手な装飾
　・影（シャドウ）を多用したUI
```

## 6. UIパーツ

### ボタン

#### メインCTA（重要なボタン）

```
形状：角丸 6px（rounded-md）
背景：マスタード（#D4A437）
文字：白（#FFFFFF）
パディング：上下14px、左右28px（py-3.5 px-7）
フォント：Noto Sans JP Bold
サイズ：16px
ホバー：背景色が少し濃く（#B8902F）
トランジション：0.2〜0.3秒
```

例：「ご相談はこちら →」「お問い合わせ →」

#### サブCTA（補助的なボタン）

```
形状：角丸 6px（rounded-md）
背景：白（#FFFFFF）
枠線：ネイビー 2px（border-2 border-navy）
文字：ネイビー（#1E3A5F）
パディング：上下14px、左右28px
フォント：Noto Sans JP Bold
サイズ：16px
ホバー：背景がネイビー薄め（#F0F4F8）
トランジション：0.2〜0.3秒
```

例：「資料請求 →」「事例を見る →」

### カード

```
形状：角丸 4px（rounded）
背景：白（#FFFFFF）
枠線：1px ライトグレー（#E0E0E0）
影：なし（フラット）
内側パディング：32px（p-8）
ホバー：罫線がネイビー（#1E3A5F）に変化
トランジション：0.2〜0.3秒
```

### リンク

```
通常：
　・色：ネイビー（#1E3A5F）
　・下線：なし

ホバー：
　・色：マスタード（#D4A437）
　・下線：出現

訪問済み：色変化なし（読み物としての一貫性）
```

### ホバー時のアニメーション

```
■ 共通方針：控えめに
　・トランジション時間：0.2〜0.3秒
　・色変化のみ（移動・拡大は基本的に使わない）

■ ボタン：背景色が少し濃く
■ カード：罫線がネイビーに
■ リンク：色変化 + 下線出現
```

### 角丸の統一

```
ボタン：6px（rounded-md）
カード：4px（rounded）
画像・動画埋め込み：4px（rounded）
タグ・バッジ：4px（rounded）
入力フォーム：4px（rounded）
```

```
■ 短いマスタード線（装飾線）
　・幅：80px〜120px

■ 長いグレー罫線
　・幅：100%（コンテナ幅）
　・高さ：1px
　・色：ライトグレー（#E0E0E0）
　・用途：大きなセクション区切り
```

## 8. レスポンシブ対応

### ブレークポイント

```
■ md: 768px（メインの切り替えポイント）
　・768px以下：スマホレイアウト
　・768px以上：PCレイアウト

■ lg: 1024px（必要に応じて）
■ xl: 1280px（必要に応じて）
```

### スマホでの変更点

```
■ ヘッダー
　・ハンバーガーメニューに切替

■ レイアウト
　・全カラムが縦積み（1カラム）に
　・最大幅は画面幅、左右余白を確保

■ フォント
　・本文：16px → 15px
　・見出し：縮小（H1: 52px → 32px など）

■ 余白
　・セクション間：80px → 60px
　・要素間：縮小
```

## 9. アクセシビリティ

### コントラスト比

```
■ 本文（#2B2B2B on #FFFFFF）：13.4:1 ✅
■ 見出し（#1E3A5F on #FFFFFF）：10.7:1 ✅
■ マスタードCTA（#FFFFFF on #D4A437）：3.2:1 ⚠️
　→ サイズを大きくして補完
```

### セマンティックHTML

```
■ 必須
　・header / nav / main / article / section / footer を適切に使う
　・H1〜H6の階層を正しく
　・alt属性を画像に必ず設定
　・装飾画像はalt=""

■ ARIA属性
　・ハンバーガーメニュー：aria-expanded
　・モーダル：aria-modal
　・ナビゲーション：aria-label
```

## 10. デザインの判断基準（迷ったとき）

このサイトのデザインで迷ったとき、以下の基準で判断してください。

### 判断基準1：「業界団体の事務局長」が見て違和感がないか

```
■ 想定読者
　・40〜60代
　・業界団体・公益法人の事務局長
　・真面目で堅実なプロフェッショナル層

■ チェックポイント
　・派手すぎないか？
　・軽すぎないか？
　・「ちゃんとした会社」に見えるか？
```

### 判断基準2：「温かみ」が出ているか

```
■ チェックポイント
　・冷たく見えないか？
　・手描き風イラストや余白で温かみが出ているか？
　・「相談していい」と思える親しみがあるか？
```

### 判断基準3：「プロフェッショナル」と「温かみ」のバランス

```
■ プロフェッショナルが強すぎたら
　・手描き風イラストを追加
　・薄いベージュの強調エリアを使う

■ 温かみが強すぎたら
　・ネイビーの面積を増やす
　・余白を整える
　・罫線でメリハリを出す
```

### 判断基準4：使う色は3+1まで

```
■ 基本3色
　・白
　・ネイビー
　・マスタード

■ +1
　・グレー（テキスト用）

■ 上記以外の色は基本使わない
　・薄いベージュ（#FAF8F2）は強調背景として例外的に使用
　・それ以外の色を増やすと、ブランド印象が崩れる
```

## 11. 参考：競合・参考にしない方向

### 参考にしない方向

```
✗ コーポレートサイト系の硬さ（IBM、富士通など）
✗ SaaS系のカラフルさ（Notion、Slackなど）
✗ メディア系の情報量（ITmedia、Forbesなど）
✗ クリエイティブ系の派手さ
```

### 目指す方向（参考）

```
✅ 信頼性のあるオウンドメディア
　・少し明朝を使ったブランディング系サイト
　・余白を活かしたシンプル構成
　・1次情報を伝える誠実なトーン
```
