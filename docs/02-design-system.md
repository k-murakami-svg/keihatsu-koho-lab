# 02. デザインシステム

## 全体トーン

**「温かみのあるプロフェッショナル」**

業界団体・公益法人の意思決定者が「ちゃんとした専門会社」と感じ、かつ「相談していい」と思える、両立したトーン。

### 軸
```
■ プロフェッショナル軸：清潔感・誠実・洗練・知的・信頼感
■ 温かみ軸：寄り添う・温かい・人の手を感じる・親しみ
```

### 避けるべき印象
- 冷たい・無機質（プロに振りすぎ）
- カジュアル・軽い（温かみに振りすぎ）
- 派手・カラフル
- 堅苦しい・古い
- スタートアップ・SaaS的なカラフルさ・グラデーション

### 用途別のフォント・サイズ指定

```
■ ヒーローのメインキャッチ
　・フォント：Noto Serif JP Bold（明朝）
　・色：ネイビー（#1E3A5F）
　・サイズ：PC 44px〜52px / スマホ 28px〜32px
　・例：「啓発に、ちゃんと手応えを。」

■ セクションのメインキャッチ
　・フォント：Noto Serif JP Bold（明朝）
　・色：ネイビー（#1E3A5F）
　・サイズ：PC 32px〜36px / スマホ 22px〜24px
　・例：「私たちが提供する3つの支援」

■ 小見出し
　・フォント：Noto Sans JP Bold
　・色：ネイビー（#1E3A5F）または マスタード（#D4A437）
　・サイズ：PC 20px〜24px / スマホ 18px〜20px

■ 本文
　・フォント：Noto Sans JP Regular
　・色：濃いグレー（#2B2B2B）
　・サイズ：PC 16px / スマホ 15px
　・行間：1.7

■ キャプション・補足
　・フォント：Noto Sans JP Regular
　・色：グレー（#666666）
　・サイズ：PC 13px / スマホ 12px

■ 英数字（数字・年号など）
　・フォント：Inter
　・色：状況に応じて
　・サイズ：本文と同じ
```

## レイアウト

### グリッドシステム

```
■ PC最大幅：1200px
　・コンテナ：max-w-[1200px] mx-auto px-6
　・左右の余白：自動センタリング

■ ブレークポイント：768px
　・768px以下：スマホレイアウト
　・768px以上：PCレイアウト
```

### 間隔（スペーシング）

```
■ セクション間隔
　・PC：80px（py-20 = 5rem = 80px）
　・スマホ：60px（py-15 = 3.75rem = 60px）

■ セクション内の要素間隔
　・PC：32px〜48px
　・スマホ：24px〜32px

■ 行間（line-height）
　・本文：1.7
　・見出し：1.4
```

### カラム構造

```
■ 基本：1カラム（読み物として上から下へ）

■ 例外
　・3つの支援セクション：3カラム
　・事例セクション：2カラム可
　・ナレッジ記事一覧：3カラム or 2カラム

■ スマホ：すべて縦積み（1カラムに変換）
```

### Tailwind CSS 設定例

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      maxWidth: {
        'container': '1200px',
      },
      spacing: {
        'section': '80px',
        'section-sm': '60px',
      },
    },
  },
}
```

## 素材方針

### 写真

```
■ 使用箇所（限定的）
　・事例ページ
　・代表メッセージページ
　・インタビュー記事

■ 使わないもの
　・フリー素材写真（信頼性を損なう）
　・トップページのヒーロー写真
```

### イラスト

```
■ 方針
　・パラパラ漫画と同じテイストの手描き風イラスト
　・サイト全体に控えめに散らす（あしらいレベル）

■ 色
　・モノクロまたはマスタード単色
　・カラフルにしすぎない

■ 使用箇所例
　・トップページのヒーロー横
　・3つの支援セクションのアイコン代わり
　・「パラパラ漫画が効く4つの理由」の各カード
　・セクション区切りのアクセント
```

### あしらい・装飾

```
■ 使うもの
　・細い罫線（マスタード または ライトグレー）でセクション区切り
　・余白で構造を作る
　・薄いベージュ背景（強調エリア）

■ 使わないもの
　・背景パターン
　・背景グラデーション
　・派手な装飾
　・影（シャドウ）を多用したUI
```

## UIパーツ

### ボタン

#### メインCTA（重要なボタン）
```
形状：角丸 6px（rounded-md）
背景：マスタード（#D4A437）
文字：白（#FFFFFF）
パディング：上下14px、左右28px（py-3.5 px-7）
フォント：Noto Sans JP Bold
サイズ：16px
ホバー：背景色が少し濃く（#B8902F）
トランジション：0.2〜0.3秒
```

例：「ご相談はこちら →」「お問い合わせ →」

#### サブCTA（補助的なボタン）
```
形状：角丸 6px（rounded-md）
背景：白（#FFFFFF）
枠線：ネイビー 2px（border-2 border-navy）
文字：ネイビー（#1E3A5F）
パディング：上下14px、左右28px
フォント：Noto Sans JP Bold
サイズ：16px
ホバー：背景がネイビー薄め（#F0F4F8）
トランジション：0.2〜0.3秒
```

例：「資料請求 →」「事例を見る →」

### カード

```
形状：角丸 4px（rounded）
背景：白（#FFFFFF）
枠線：1px ライトグレー（#E0E0E0）
影：なし（フラット）
内側パディング：32px（p-8）
ホバー：罫線がネイビー（#1E3A5F）に変化
トランジション：0.2〜0.3秒
```

### リンク

```
通常：
　・色：ネイビー（#1E3A5F）
　・下線：なし

ホバー：
　・色：マスタード（#D4A437）
　・下線：出現

訪問済み：色変化なし（読み物としての一貫性）
```

### ホバー時のアニメーション

```
■ 共通方針：控えめに
　・トランジション時間：0.2〜0.3秒
　・色変化のみ（移動・拡大は基本的に使わない）

■ ボタン：背景色が少し濃く
■ カード：罫線がネイビーに
■ リンク：色変化 + 下線出現
```

### 角丸の統一

```
ボタン：6px（rounded-md）
カード：4px（rounded）
画像・動画埋め込み：4px（rounded）
タグ・バッジ：4px（rounded）
入力フォーム：4px（rounded）
```

## 区切り線・あしらい

### セクション区切り

```
■ 短いマスタード線（装飾線）
　・幅：80px〜120px
　・高さ：3px〜4px
　・色：マスタード（#D4A437）
　・配置：セクションタイトルの下、中央配置

■ 長いグレー罫線
　・幅：100%（コンテナ幅）
　・高さ：1px
　・色：ライトグレー（#E0E0E0）
　・用途：大きなセクション区切り
```

## レスポンシブ対応

### ブレークポイント

```
■ md: 768px
　・基本的な切り替えポイント
　・768px以下：スマホレイアウト
　・768px以上：PCレイアウト

■ lg: 1024px（必要に応じて）
　・タブレット
　・大画面スマホ

■ xl: 1280px（必要に応じて）
　・大画面PC
```

### スマホでの変更点

```
■ ヘッダー
　・ハンバーガーメニューに切替

■ レイアウト
　・全カラムが縦積み（1カラム）に
　・最大幅は画面幅、左右余白を確保

■ フォント
　・本文：16px → 15px
　・見出し：縮小（H1: 52px → 32px など）

■ 余白
　・セクション間：80px → 60px
　・要素間：縮小
```

## アクセシビリティ

```
■ コントラスト比
　・本文（#2B2B2B on #FFFFFF）：13.4:1 ✅
　・見出し（#1E3A5F on #FFFFFF）：10.7:1 ✅
　・マスタードCTA（#FFFFFF on #D4A437）：3.2:1 ⚠️
　　→ サイズを大きくして補完

■ セマンティックHTML
　・header / nav / main / article / section / footer を適切に
　・H1〜H6の階層を正しく

■ alt属性
　・画像にはすべてalt属性
　・装飾画像はalt=""

■ ARIA属性
　・ハンバーガーメニュー：aria-expanded
　・モーダル：aria-modal
　・ナビゲーション：aria-label
```
